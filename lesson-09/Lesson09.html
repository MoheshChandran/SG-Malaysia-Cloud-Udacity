Lesson 09 - Working with Remotes
how to create remote repositories on GitHub and how to get and send changes to the remote repository

Remote Repository

Git is a distributed version control system which means there is not one main repository of information.Remote repositories are compared to local repositories.

We can collaborate with our friends like the project you see now.And Remote Repository make our code have a copy make sure more safety.

GitHub is one of popular remote repository provider.


Add A Remote Repository

  • git remote -v

    • use to check the remote repository on your local repository.

  • git remote add

    • use to add the remote repository on your local repository.

    Example git remote add origin https://github.com/richardkalehoff/RichardsFantasticProject.git

    Also we can change origin to other words if you like.


    Push Changes To A Remote

  • git log --oneline --graph --decorate --all

    • use to show the detail log of local repository.

  • git push 'remote-shortname' 'local-branch'

    • use to upload the specified local branch to remote repository like GitHub.

    Example git push origin master


    Pulling Changes From A Remote

  • git pull origin master

    • use to donwload the specified local branch of remote repository like GitHub to local repository.

    if local repo is behind of remote repo. pull will download and make both same.

  • the commit(s) on the remote branch are copied to the local repository
  • the local tracking branch (origin/master) is moved to point to the most recent commit
  • the local tracking branch (origin/master) is merged into the local branch (master)

    • If you don't want to automatically merge the local branch with the tracking branch then you wouldn't use git pull you would use a different command called git fetch.


    Pull vs Fetch

  • git fetch origin master

    • git fetch as half of a git pull.

    Git fetch is used to retrieve commits from a remote repository's branch but it does not automatically merge the local branch.

    In this case,we can avoid merge conflict in git pull
