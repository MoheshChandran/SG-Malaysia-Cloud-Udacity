Git Tag

  • git tag – Listing all the existing tag in your repository (with optional -l or --list), 2. also can list all the tag with matching pattern
  • Creating tags. There is two types of tags : lightweight and annotated
    • git tag “tag name” : this is lightweight tag
    • git tag -a “tag name” -m "tag message" : this is annotated tag, where need to supply -a, -s or -m options
    • You can also tag on past commits by using this command : git tag -a “tag name” “commit sha id”
  • git tag : used to display all of the tag that are in the repository
  • git tag -d “tag name” – this command is use to delete the specific tag from the repository

Git Branch

  • git branch : is use to list all the branch names in the repository
  • git branch “branch name” : this command is used to create a branch
  • git checkout “branch name” : this command is used to switch branch , So this will remove all of the files that are referenced by commits in the current branch. It will replace them with the files that are referenced by the commits in the reference branch.
  • git branch -d “branch name” : is used to delete branch from the repo

Git Merge

  • git merge : The git merge command lets you take the independent lines of development created by git branch and integrate them into a single branch. Git merge will combine multiple sequences of com-mits into one unified history.
    • $ git merge < name-of-branch-to-merge-in >
