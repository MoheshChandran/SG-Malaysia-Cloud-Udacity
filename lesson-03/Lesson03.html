What is Version Control

Software that helps you control (or manage) the different versions...of something (typically source code). Also known as revision control or source control

There are 2 main types of version control system models:

  • centralized model - All users connect to a central, master repository. Eg. CVS, Subversion (or SVN) and Perforce
  • distributed model - Each user has the entire repository on their computer Eg. Git, Mercurial, and Bazaar.

Some text editor, such as Google Doc provide revision history to keep track of changes/version the user made, but the functionality is limited (eg. not able to label changes)

Git Terminology

Here is few common Git terms that is often used

  • Version Control System (VCS)
    Tool that manages different versions of source code. Also known as source code manager (SCM)
  • Commit
    A snapshot reference that saves the project's files and information
  • Repository / repo
    directory which contains your project work, as well as a few files (hidden by default on Mac OS X). Repositories can exist either locally on your computer or as a remote copy on another computer. A repository is made up of commits
  • Working Directory
    Files that you see in the current local repository.
  • Checkout
    Content in the repository has been copied to the Working Directory.
  • Staging Area / Staging Index / Index
    A file in the Git directory that stores information about what will go into your next commit. Files on the Staging Index are poised to be added to the repository
  • SHA (Secure Hash Algorithm)
    A 40-character length (0–9 and a–f) commit ID string. It is calculated based on the contents of a file or directory structure in Git
  • Branch
    A new line of development that diverges from the main line of development, that allows development continue without altering the main line
In Git repository, all changes made to files are pushed/moved into Staging Area, and once committed save to the repository

Git Configuration

Here is the one-time configurations needed before starting using Git

# sets up Git with your name
git config --global user.name ""
# sets up Git with your email
git config --global user.email ""
# makes sure that Git output is colored
git config --global color.ui auto
# displays the original state in a conflict
git config --global merge.conflictstyle diff3 git config --list
# Choose either one code editor (optional) - atom, sublime text or vscode
# For Atom setup
git config --global core.editor "atom --wait"
# For Sublime Text setup
git config --global core.editor "'C:/Program Files/Sublime Text 2/sublime_text.exe' -n -w"
# For VSCode setup
git config --global core.editor "code --wait"
