Creat a Pull Request

  • A pull request is a request to the original or source repository's maintainer to include changes in their project that you made in your fork or their project.
  • Prior to the send pull request to the group or open-source projects, make sure you check the below
    1. reviewing the project's CONTRIBUTING.md file
    2. checking out the project's existing issues
    3. talking with the project maintainer

Stay in sync with source project

  • Star: to keep track of repositories (the button is located in the right-up side of the github repository webpage.)
  • Watch: also useful to keep track of the repositories. Unlike adding 'star' this is not really related to the populatity of the project.
  • "git remote add upstream " - upstream as the shortname to reference the source repository.
  • "git fetch upstream master" - to retrieve upstream changes.

Manage an active PR

  • Make use of "Conversation" tab in GitHub to communicate back and forth with the project's maintainer to clarify exactly what they want!
  • Also, as most developers participating in open source projects are doing it unpaid, therefore, remeber to:
    • be kind
    • be patient - project maintainer will repond as soon as they are able to!

Squash Commits

  • "git rebase" command, to squash commits altogether!
  • "git rebase" command will move commits to have a new base.
  • "git rebase -i HEAD~3" : This command line is telling Git to use "HEAD~3" as the base where all of the other commits will connect to.
  • Also, "-i" here mans "interactive", to use rebase command in 'interactive mode'.
