git init

create a new empty git repository. Create a hidden .git directory that records all of the commits.

mkdir new-git-project # create new directory
cd new-git-project # change directory
git init # initialize git repo

Here is the list of files/directories inside .git directory. Recommended not to change/edit these files

  • config file
    project specific configuration settings. These values are specific to that single repository.
  • description file
    this file is only used by the GitWeb program (can be ignored)
  • hooks directory
    client-side or server-side scripts that used to hook into Git's different lifecycle events
  • info directory
    contains the global excludes file
  • objects directory
    directory that store all of the commits
  • refs directory
    directory that holds pointers to commits (basically the "branches" and "tags")

git clone

create an identical copy of an existing remote repository into a new directory, same name with the repo

git clone https://github.com/udacity/course-git-blog-project # create a clone inside directory course-git-blog-project
git clone https://github.com/udacity/course-git-blog-project blog-project # create a clone inside directory blog-project

git status

  • new files that have been created in the Working Directory that Git hasn't started tracking, yet
  • files that Git is tracking that have been modified
