Lesson 22 - Servers and Security Groups
Learn Security Group, Ingress/Egress rules, Auto Scaling Concept

Security Group

Security Groups are associated with EC2 instances and provide security at the protocol and port access level. Each security group, working much the same way as a firewall -contains a set of rules that filter traffic coming into and out of an EC2 instance

Security groups are specific to individual resources (EC2 servers, databases) and not to subnets

Ingress rules:

  • Ingress rules are for inbound traffic
  • Ingress rules restrict or allow traffic trying to reach our resources on specific ports
  • For ingress rules, we want to limit inbound traffic, for security, to a single port or just a handful of ports required by the application we are running

Egress rules:

  • Egress rules are for outbound traffic
  • Egress rules restrict or allow traffic originating from our server
  • For egress rules, we want to give the resource full access to the internet, so we give egress access to all ports, from 0 all the way to 65535

Traffic is blocked by default

Auto Scaling Concepts

Monitors your applications and automatically adjusts capacity to maintain steady, predictable performance at the lowest possible cost

  • Scaling Policy

    A Scaling Policy is the criteria used to decide when to Add or Remove Servers from your Auto Scaling Group

  • Launch Configuration

    A launch configuration is an instance configuration template that an Auto Scaling group uses to launch EC2 instances

  • Load Balancer

    A load balancer allows you to reduce your Auto Scaling down and Scale up easily, and user will not feel any different

Adding Target Groups and Listeners

A target group tells a load balancer where to direct traffic to : EC2 instances, fixed IP addresses; or AWS Lambda functions, amongst others

A listener is a process that checks for connection requests, using the protocol and port that you configure. The rules that you define for a listener determine how the load balancer routes requests to the targets in one or more target group

Health Checks are the requests your Application Load Balancer periodically sends to its registered targets
